# User Profile

The user profile page displays key information about the currently logged-in user, including basic details, user posts, and connected accounts.

---

## Data Included

The profile page fetches user data from the database using Prisma, including:
- User details (name, email, etc.).
- **Posts**: All posts created by the user.
- **Connected Accounts**: List of third-party OAuth provider accounts associated with the user.

The data retrieval now uses:

```ts
const user = await prisma.user.findUnique({
  where: { id: session.user.id },
  include: {
    posts: true,
    accounts: true,
  },
});
```

---

## Connected Accounts

A new section, "Connected Accounts", now appears on the profile page.

- Each connected account is listed with its provider name and provider account ID.
- If the user has no connected accounts, a message is shown:  
  *No connected accounts.*

Sample UI:

```tsx
<div className="bg-white p-6 rounded-lg shadow">
  <h2 className="text-xl font-semibold mb-4">Connected Accounts</h2>
  {user.accounts && user.accounts.length > 0 ? (
    <ul className="list-disc pl-5 space-y-2">
      {user.accounts.map((account) => (
        <li key={account.id}>
          {account.provider}: {account.providerAccountId}
        </li>
      ))}
    </ul>
  ) : (
    <p className="text-gray-500">No connected accounts.</p>
  )}
</div>
```

## API Endpoint: Get User by ID

A new API route provides detailed user data, including posts and connected accounts:

**Endpoint:**  
`GET /api/users/[id]`

**Description:**  
Returns user information (including their posts and connected accounts) for the specified user ID.

**Example Request:**

```http
GET /api/users/1234
```

**Example Response:**

```json
{
  "id": "1234",
  "name": "Alice Example",
  "email": "alice@example.com",
  "posts": [
    { "id": "a1", "title": "First Post", ... }
    // ...
  ],
  "accounts": [
    { "id": "acc1", "provider": "google", "providerAccountId": "abc123", ... }
    // ...
  ]
}
```

**Error Responses:**
- `404 Not Found` if the user does not exist
- `500 Internal Server Error` for generic failures

**Sample Implementation:**

```ts
import { NextResponse } from 'next/server';
import { prisma } from '../../../../lib/prisma';

export async function GET(
  request: Request,
  { params }: { params: { id: string } }
) {
  try {
    const { id } = params;
    const user = await prisma.user.findUnique({
      where: { id },
      include: {
        posts: true,    // Include posts by the user
        accounts: true, // Include user accounts
      },
    });

    if (!user) {
      return NextResponse.json({ message: 'User not found' }, { status: 404 });
    }

    return NextResponse.json(user);
  } catch (error) {
    console.error('Error fetching user:', error);
    return NextResponse.json({ message: 'Error fetching user' }, { status: 500 });
  }
}
```
